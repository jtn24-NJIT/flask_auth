Containers: A standard unit of software, a docker container packages up code and all of its dependencies to ensure efficient, reliable and quick running of said application from one computing environment to others. Thanks to docker, a container allows for easy execution of a package of software as well as everything needed to run it (IE: system tools, system libraries, settings, etc.). These containers in turn allow for quicker and smaller ways to run applications without the typical usage of a virtual machine which requires guest operating systems and a hypervisor (in comparison to a container engine and singular operating system for docker containers).




Images: Automatically generated when a docker container is made, a docker image is a read-only template that contains a set of instructions for making an application run on docker. When ran, it can become one of many instances of a container. Made up of multiple layers (which build off from the previous layer with some changes), one of their advantages is that they are reusable, which means they can be deployed on any host and across multiple projects (thus saving time). Similar to a git commit, a docker image is like a snapshot of a docker container... but they are also immutable, making it so new software or configuration does not change the image.





Docker Hub Link for Project: https://hub.docker.com/repository/docker/jtn24/first-website

How to Run Project:
1. docker pull jtn24/first-website
2. docker images (use it to see image ID)
3. docker run -d -p 8080:80 imageID
4. docker ps (shows container ID)
5. docker exec -it containerID /bin/bash (opens a bash shell into the container)

Docker Commands

Docker Pull Command: docker pull user/image, essentially the same as a git pull (gets an image to local from docker hub).

Docker Push Command: docker push user/image:tag, essentially the same as a git push (uploads an image from local to docker hub), often times the tag is the version being pushed.

Docker Images Command: docker images, essentially shows all currently running images.

Docker rmi Command: docker rmi image_id/image_name, removes an image as specified.

Docker ps Command: docker ps, shows information about currently running images.

Docker commit Command: docker commit container_id image_name:tag, allows for creation of an image from a container.

Docker start Command: docker start container_id/container_name, starts running a container.

Docker exec Command: docker exec containerID /bin/bash, runs a command in a running container.

Docker stop Command: docker stop container_id/container_name, stops a running container.
